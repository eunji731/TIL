# 파일 전송 프로토콜 정리 (FTP / SFTP / SCP)

## 1. FTP (File Transfer Protocol)
- 가장 오래된 파일 전송 프로토콜
- 암호화되지 않음(계정/비밀번호 평문 전송)
- 기본 포트: 21
- 데이터 포트와 제어 포트를 구분하여 사용
- 외부 공격 위험이 있어 현재 실무에서는 거의 사용하지 않음

### 특징
- 빠르지만 보안성이 매우 낮음
- 익명 로그인(anonymous) 방식도 존재
- PASV/PORT 모드 개념 존재

---

## 2. SFTP (SSH File Transfer Protocol)
- SSH 위에서 동작하는 파일 전송 프로토콜
- 포트 22 사용
- 전송 과정 전체가 암호화됨
- 디렉터리 열기, 목록 조회, 삭제, 권한 변경 가능

### 특징
- SSH 세션 안에서 동작 → 매우 안전함
- FTP와 달리 단일 채널로 통신
- 전송 재개(Resume) 가능
- FTP와 이름만 비슷하고 내부 구조는 완전히 다름

---

## 3. SCP (Secure Copy Protocol)
- SSH 기반의 "단순 복사" 프로토콜
- 포트 22 사용
- 명령 한 번으로 파일을 통째로 복사

### 특징
- SFTP보다 단순하고 빠름
- 디렉터리 탐색, 목록 조회 기능 없음
- 재전송/Resume 기능 없음
- 실무에서는 빠른 단방향 전송으로 자주 활용

---

## 4. 차이 비교

| 항목 | FTP | SFTP | SCP |
|------|------|------|------|
| 보안 | 없음 | 강함(SSH) | 강함(SSH) |
| 기능 | 파일 전송 | 파일 전송 + 파일 관리 | 단순 복사 |
| 포트 | 21 | 22 | 22 |
| 재전송(Resume) | X | O | X |
| 사용성 | 과거식 | 현재 표준 | 빠른 이동에 적합 |

---

## 5. 결론
- FTP는 비보안이라 거의 사용하지 않음  
- SFTP는 “보안 파일 시스템”처럼 사용  
- SCP는 “빠른 보안 복사”만 필요할 때 사용
